BeConnect

Proyecto – Ingeniería Web I

Planteamiento del Proyecto

Actualmente muchos trabajadores independientes enfrentan dificultades para promocionar sus servicios y establecer contacto directo con posibles clientes. A su vez, las personas que requieren un servicio específico suelen tener problemas para encontrar prestadores confiables y disponibles en su zona.

Ante esta necesidad, se plantea el desarrollo de un sitio web diseñado como un espacio de intermediación entre emprendedores y clientes. En la plataforma los emprendedores podrán registrarse para ofrecer sus servicios, mientras que los clientes podrán buscar y solicitar el servicio que requieren de manera rápida y sencilla.

En su fase inicial, la página web se centrará en un conjunto limitado de categorías, tales como belleza y tecnología, con el objetivo de optimizar su implementación inicial. Posteriormente, la plataforma podrá ampliarse e incorporar nuevos sectores de servicio.

Justificación

Desde una perspectiva socioeconómica, este proyecto responde a la falta de plataformas accesibles que permitan la conexión directa entre prestadores de servicios independientes y clientes. La propuesta busca fortalecer el emprendimiento, fomentar el autoempleo y facilitar el acceso a servicios confiables, contribuyendo al desarrollo económico local.

Desde el ámbito social, la plataforma favorece la visibilidad de pequeños emprendedores, permitiéndoles ampliar su alcance y generar nuevas oportunidades laborales. El proyecto incorpora técnicas de desarrollo web moderno, diseño de interfaces y programación, además del consumo de APIs externas y la adopción de estándares y buenas prácticas de la W3C.

Finalmente, se alinea con el Objetivo de Desarrollo Sostenible (ODS) 8: Trabajo decente y crecimiento económico, al apoyar el emprendimiento y la generación de ingresos, y con el ODS 9: Industria, innovación e infraestructura, al emplear la tecnología como herramienta de conexión e innovación.

Objetivo General → Anterior

Diseñar e implementar una plataforma web de tipo SPA que funcione como un medio de conexión entre emprendedores prestadores de servicios y clientes, orientada a centralizar la oferta y la demanda de servicios en las áreas de belleza y tecnología, facilitando procesos como el registro de usuarios, búsqueda y solicitud de servicios, mediante una interfaz accesible, responsiva y fácil de usar.

Objetivo General → Corregido

Diseñar un sitio web que funcione como un medio de conexión entre emprendedores prestadores de servicios y clientes, orientado a centralizar la oferta y la demanda de servicios en las áreas de belleza y tecnología, con el fin de fortalecer la presencia y crecimiento de los emprendedores, facilitando que sus servicios sean más visibles y accesibles para los usuarios.

Objetivos Específicos → Anterior

  • Integrar una API externa que permita complementar la información del sistema, incorporando datos de ubicación, contacto u otra información dinámica relevante.
  • Determinar criterios de diseño responsivo y accesibilidad web que aseguren el correcto funcionamiento del sitio en diferentes dispositivos, considerando aspectos como contraste adecuado y ajuste de tamaño de fuente.
  • Aplicar buenas prácticas de desarrollo web mediante el uso de control de versiones con Git y GitHub, con el fin de mantener un proceso de desarrollo organizado y controlado.
  • Evaluar el funcionamiento general de la plataforma para verificar el cumplimiento de los objetivos propuestos.
  • Definir la estructura funcional de la aplicación web, estableciendo los roles de usuario y los flujos de interacción entre clientes y prestadores de servicios.

Objetivos Específicos → Corregido

  • Diseñar la maqueta visual del sitio web para representar la interfaz gráfica y la experiencia de usuario, utilizando herramientas de prototipado y criterios de diseño responsivo.
  • Diseñar la estructura y funcionamiento del sitio web para organizar los módulos, roles de usuario y procesos del sistema, mediante modelos y diagramas de diseño antes de la etapa de desarrollo.
  • Analizar el funcionamiento del sitio web para identificar posibles fallas y asegurar el cumplimiento de los objetivos planteados.
  • Organizar el desarrollo del proyecto mediante control de versiones para mantener un registro estructurado de los cambios realizados, utilizando Git y GitHub durante todo el proceso de desarrollo.
  • Integrar una API externa que permita complementar la información del sistema mediante la incorporación de datos dinámicos relevantes.
